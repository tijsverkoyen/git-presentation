Git via CLI/Terminal

Er zijn heel wat grafische clients (vb: Sourcetree) maar daardoor weet je eigenlijk niet wat je doet.

Wij gaan enkele basis commando's doen, zo gaan we ook verstaan wat je doet.

git init

initialize a repository

Git is lokaal, je kan dus perfect lokaal starten zonder internet. Hiermee initialiseer je dus een nieuwe Git repo

git status

What is the current state?

Dit commando gebruik je zeer veel. Mag je ook oneinding veel doen.

Create a file:
touch 1.txt

git status

Untracked is een file die niet in Git zit.

git add 1.txt

Stage a file by adding it.

Een file toevoegen, zodat die getracked gaat worden bij de volgende commit. Lees: een file toevoegen aan een commit.

git status

New file is een file die bij de volgende commit zal getracked worden

git commit -m
"Our 1st commit"

Store staged changes

Een commit is een verzameling van changes.

Hou je commits klein.

Create some files:


          touch 2.txt
          touch 3.txt
          mkdir foo
          touch foo/4.txt

git add *

Stage all

Je kan ook wildcards gebruiken, vb: *.txt

git commit -m
"Add some files"

git status

git log

Show history

Lijst alle commits op, je kan hier eventueel ook details zien via git show $hash

git remote add
origin

Hiermee laat je weten welke "centrale" server we gaan gebruiken.

git push -u origin master

Jouw lokale wijzigingen naar Github pushen.

git pull origin master

Git Flow

"Invented" by Vincent Driessens

Is een concept, geen zaken nodig. Al zijn er tools om het af te dwingen.

Some special branches

  • master == production
  • staging == staging

De code in deze twee branches moet stabiel zijn, kan immers op elk moment gedeployed worden

Staging is eigenlijk wat nog niet op productie staat, kan dus al nieuwe features bevatten.

New feature or bugfix
==
branch

  1. Create a branch
    git co -b new-feature
  2. Develop & Commit
    git add ...
    git commit ...
  3. Push the branch
    git push
  1. Create a merge request
  2. Code review by collegue
  3. Process feedback (if needed)
    git add ...
    git commit ...
    git push
  1. Merge
  2. pull
    git checkout staging
    git pull
  3. Merge in master branch
    git checkout master
    git merge staging
  4. Deploy
